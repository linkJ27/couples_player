# 隐私与安全边界

## 产品边界

应用只做本地媒体播放与同步控制，不提供以下能力：

- 视频下载。
- 资源站搜索。
- 网盘解析。
- DRM 绕过。
- 默认视频上传或转发。

## 数据最小化

服务端可见：

- 房间码。
- 临时 sessionId。
- 在线状态。
- WebRTC 信令。
- 房间当前播放状态。

服务端不应接收：

- 本地完整文件路径。
- 视频文件内容。
- 字幕文件内容，除非用户明确发送。
- 永久观影历史，除非后续加入账号体系并明确授权。

## 文件信息

房间内可以交换：

- 展示文件名。
- 文件大小。
- 时长。
- 编码信息。
- 分段指纹。

文件路径只保存在本机。

## 加密

- WebSocket 使用 TLS。
- WebRTC 使用 DTLS/SCTP。
- 后续如要降低服务端可见性，可在房间内增加端到端加密层。

## 滥用控制

- 房间码短期有效。
- 限制房间人数，第一版固定 2 人。
- 限制消息大小和发送频率。
- 表情反应限流。
