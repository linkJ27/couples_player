# ADR 0003：本地媒体身份识别

## 状态

Accepted

## 背景

双方本地文件路径不同，文件名也可能不同。只靠文件名容易误匹配，只靠全文件 hash 对大视频成本高。

## 决策

采用三级匹配：

1. 快速元数据：文件名、大小、时长、分辨率、音轨数量。
2. 分段指纹：头部、中段、尾部多个 chunk hash。
3. 严格校验：用户手动触发全文件 hash。

文件路径只保存在本机，房间只交换显示名、大小、时长、媒体信息和指纹。

## 理由

- 快速元数据能提供即时反馈。
- 分段指纹在速度和准确性之间平衡。
- 全文件 hash 保留给高确定性场景。
- 不发送路径能减少隐私泄漏。

## 影响

- UI 需要表达“可能匹配”和“严格匹配”的区别。
- 指纹算法需要跨平台稳定。
- 后续播放列表应以 `mediaId` 为核心，而不是以文件路径为核心。
