# ADR 0004：播放状态权威模型

## 状态

Accepted

## 背景

同步播放最容易出现的问题是双方同时控制、网络延迟、进度漂移和断线恢复。如果只广播当前播放秒数，会产生频繁抖动和状态覆盖。

## 决策

播放状态建模为房间权威时间轴：

- `anchorMediaTimeMs`
- `anchorRoomTimeMs`
- `playbackRate`
- `state`
- `epoch`
- `version`

客户端通过本地单调时钟推算目标播放进度，并根据漂移阈值微调或 seek。

## 理由

- 时间轴模型比持续广播秒数更稳定。
- `epoch` 能隔离重连、切集、主控切换后的旧消息。
- `version` 能处理命令顺序。
- 小漂移用倍速修正能避免频繁 seek 的观感问题。

## 影响

- 客户端需要实现房间时间估算。
- 播放器 adapter 需要支持可靠读取进度和设置临时倍速。
- 测试必须覆盖延迟、乱序、断线和重连。
